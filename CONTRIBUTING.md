# Contributing to Depureco MCP

Thanks for your interest in the Depureco Industrial Vacuum MCP Server!

## How to contribute

### Reporting issues

If you find a bug, unexpected behavior, or have suggestions for improvements, please open an issue using one of our templates:

- **Bug report** — for problems with the MCP endpoint or integration examples
- **Feature request** — for new tools, parameters, or capabilities
- **Documentation improvement** — for unclear or missing documentation

### Submitting pull requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test your integration examples
5. Update documentation if applicable
6. Submit a pull request with a clear description

### Code style for examples

- **Python**: PEP 8 compliant, type hints where useful
- **JavaScript**: Modern ES6+, async/await preferred
- **Curl/Bash**: POSIX-compliant when possible

### Adding integration examples

We welcome integration examples for additional frameworks and languages:

- LangChain, LlamaIndex, AutoGen, CrewAI
- Ruby, Go, Rust, PHP, Java
- Browser-based JavaScript
- Custom MCP clients

Place your example in the appropriate `examples/` subdirectory with a brief README explaining the use case.

### Reporting documentation gaps

If you find documentation that is unclear, outdated, or missing, please open an issue with the `documentation` label. We particularly welcome contributions in:

- ATEX zone classification scenarios
- Filter class selection guidance
- Industry-specific use cases
- Multilingual documentation improvements

## Code of conduct

Be respectful, constructive, and professional. We are building tools for industrial safety applications — quality and accuracy matter.
